# 📊 UPI Transaction Failure Analysis

## 🚀 Project Overview
This project analyzes UPI transaction data to identify patterns in transaction failures and provide actionable business insights.

## 🧠 Problem Statement
Why are UPI transactions failing, where is the issue occurring, and what can be done to reduce failures?

---

## 🛠 Tools Used
- Excel (Data Cleaning & Pivot Analysis)
- SQL (Data Querying)
- Python (EDA & Visualization)
- Power BI (Dashboard)

---

## 📊 Key Insights

- 🔴 Failure rate is around 15% of total transactions
- 🏦 Axis Bank has the highest number of failures
- ⏰ Failures are higher during non-peak hours
- ⚠️ Server error and timeout are major failure reasons

## 💡 Recommendations

- Improve system performance during non-peak hours
- Optimize bank integrations (especially Axis)
- Reduce server and timeout errors
- Encourage retry mechanisms for failed transactions
